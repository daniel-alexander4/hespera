Features

Turn optional behaviors on or off
When a scan finds a video file with container corruption (bad framing that can glitch playback), Hespera losslessly rewrites it (remux, no re-encode), verifies the result, and atomically replaces the original — no re-encode, no quality loss. Corruption that damages the actual video data can't be repaired and is only flagged. This is the one operation that writes back to your media files; turn it off to detect and flag only. Run a deep check any time from the Libraries page.
Watch the library folders and run the usual scan chain automatically when files are added, moved, or removed — new media appears without clicking Scan. Debounced, so a copy in progress doesn't trigger mid-write; a scan already running is never doubled. Turn off to scan only via the Libraries page.
Off (default): the now-playing page shows no lyrics and the album cover expands to fill the space. On: Hespera fetches synced (karaoke) lyrics per track from the free LRCLIB service and scrolls them under the cover. Tracks with no synced lyrics still hide the card. Takes effect on the next now-playing view.
Off (default). On: once per session the topbar version pill checks the newest published release — green means you're on the latest, red means an update exists (click the pill to download it). Off, the pill stays yellow (status unknown); clicking it still checks on demand. Nothing installs automatically.

Subtitles

Subtitle defaults and appearance; default audio language
Off (default): playback starts with subtitles off. On: a text subtitle track is enabled automatically when the file has one — picking the subtitle language below when set. Image-based (burn-in) subtitles are never auto-enabled, and switching subtitles off in the player always wins.
With subtitles on by default: only a text track in this language is auto-enabled (no match — subtitles stay off). Blank: any text track qualifies. A code like en, de, ja — three-letter forms like eng work too.
Size, background, and position apply to text subtitles rendered by the player. Image-based (burn-in) subtitles are part of the video picture and can't be restyled.

Audio

When a file has several audio tracks, prefer the one in this language. Blank: use the file's own default track. Tracks without a language tag are unaffected.

About

Third-party data sources and open-source attributions

Hespera is built on data and software from the projects below. All third-party attributions live here.

The Movie Database (TMDB)

This product uses the TMDB API but is not endorsed or certified by TMDB.

TV and movie metadata, artwork, and cast/people data are sourced from TMDB.

Music data & APIs

  • MusicBrainz — album/artist matching data (core data released under CC0).
  • Cover Art Archive — album cover art.
  • ListenBrainz — track popularity and similar-artist data.
  • LRCLIB — synced lyrics.
  • fanart.tv — artist images (optional, user-keyed).
  • TheAudioDB — artist bios/images (optional, user-keyed).
  • Last.fm — additional track popularity data for Shuffle Most Popular (optional, user-keyed).

Other sources

  • OpenSubtitles — on-demand subtitle search/download (optional, user-keyed).
  • Wikipedia / Wikimedia Commons — artist/person biographies and images, used under CC BY-SA.
