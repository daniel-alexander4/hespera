  <action>
Create `internal/match/pipeline_integration_test.go` (package match) with two integration tests. Use the existing `openTestDB(t)` helper from `dedup_test.go`.

**Test helper -- `newMockMusicServer(t)`:**
Create a single `httptest.Server` that routes requests by URL path to return canned JSON responses. This one server handles all five external services (MusicBrainz, Wikipedia, Wikidata, Wikimedia Commons, Cover Art Archive).

Path routing:
- `/ws/2/artist?query=...` -> artist search response (returns 1 artist with ID "test-artist-mbid", score 100, name matching query)
- `/ws/2/artist/test-artist-mbid?inc=url-rels...` -> artist lookup with relations pointing back at the mock server:
  - wikipedia relation URL: `{mockURL}/wiki/Test_Artist`
  - wikidata relation URL: `{mockURL}/wiki/Q12345`
- `/ws/2/release-group?query=...` -> release group search (returns 1 result: id "test-rg-id", title "Dark Side of the Moon", artist "Pink Floyd", score 100, primary-type "Album", first-release-date "1973-03-01", with a release id "test-release-id")
- `/api/rest_v1/page/summary/Test_Artist` -> Wikipedia summary (extract: "Test artist biography text.")
- `/wiki/Special:EntityData/Q12345.json` -> Wikidata entity JSON with P18 claim (filename: "Test_artist.jpg") and enwiki sitelink
- `/wiki/Special:FilePath/Test_artist.jpg?width=500` -> return small PNG bytes (Content-Type: image/jpeg)
- `/release-group/test-rg-id` -> CAA response with one front image, thumbnail large URL pointing at `{mockURL}/caa-image.jpg`
- `/caa-image.jpg` -> return small JPEG bytes (Content-Type: image/jpeg)
- All other paths -> 404

**Test helper -- `newTestMatcher(t, db)`:**
Creates a `Matcher` with:
- `db`: the passed-in test DB
- `dataDir`: `t.TempDir()`
- `mb`: `&MBClient{client: srv.Client(), baseURL: srv.URL + "/ws/2", lastReq: time.Time{}}` (lastReq zero bypasses throttle)
- `caa`: `&CAAClient{client: srv.Client(), baseURL: srv.URL, thumbDir: filepath.Join(dataDir, "thumbs", "music")}`

Note: Setting `lastReq` to `time.Time{}` means `time.Since(lastReq)` returns a very large duration, so `throttle()` never sleeps. This bypasses the 1 req/sec rate limiter.

**Test 1 -- `TestRunMusicMatchIntegrationHappyPath(t *testing.T)`:**
1. Create DB via `openTestDB(t)`.
2. Insert a library (type "music"), an artist ("Pink Floyd", no mbid/bio/art), an album ("Dark Side of the Moon", year 1973, match_status ""), a track (satisfies FK), and a scan_jobs row (job_type "music_match", status "running").
3. Create matcher via `newTestMatcher(t, db)`.
4. Call `m.RunMusicMatch(ctx, jobID, libID)` -- should succeed.
5. Verify artist enrichment:
   - Query `music_artists WHERE id=artistID`: `musicbrainz_id` should be "test-artist-mbid"
   - `bio` should contain "Test artist biography text"
   - `art_path` should be non-empty (file should exist on disk)
6. Verify album matching:
   - Query `music_albums WHERE id=albumID`: `match_status` should be "matched" (score >= 70)
   - `musicbrainz_id` should be "test-rg-id"
   - `match_confidence` should be > 0
   - `match_source` should be "musicbrainz"
7. Verify cover art: `art_path` on album should be non-empty, and the file should exist on disk.
8. Verify job progress: `scan_jobs WHERE id=jobID` should have `progress_total=1` and `progress_current=1`.

**Test 2 -- `TestRunMusicMatchIntegrationPartialFailure(t *testing.T)`:**
Test that artist enrichment failure does not prevent album matching.
1. Create a mock server that returns 500 for `/ws/2/artist?query=...` (artist search fails) but returns normal responses for release-group search and CAA.
2. Seed DB same as happy path (artist without mbid, album with empty match_status).
3. Call `RunMusicMatch` -- should NOT return error (per-artist errors are non-fatal).
4. Verify artist: `musicbrainz_id` should still be empty (enrichment failed).
5. Verify album: `match_status` should be "matched" or "uncertain" (album matching succeeded independently).

Use inline JSON fixture strings (not files). Keep fixture data minimal but structurally correct. Use `t.Run` subtests for each verification assertion within each test.
