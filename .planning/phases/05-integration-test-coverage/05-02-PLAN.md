Create a helper in the test file that makes a closed channel: `ch := make(chan time.Time); close(ch)` -- reading from a closed channel returns the zero value immediately and repeatedly. This avoids changing NewClient.

Create `internal/tmdb/matcher_integration_test.go` (package tmdb) with two integration tests.

**Test helper -- `openTestDB(t)`:**
Same pattern as `internal/match/dedup_test.go`: create temp SQLite in `t.TempDir()`, call `isodb.Open` + `isodb.Migrate`, register cleanup.

**Test helper -- `newMockTMDBServer(t)`:**
A single `httptest.Server` routing by URL path:
- `/3/search/tv?...` -> JSON response with one result: id 1396, name "Breaking Bad", first_air_date "2008-01-20", poster_path "/poster.jpg", popularity 234.5
- `/3/tv/1396?...` -> JSON show response: id 1396, name "Breaking Bad", poster_path "/poster.jpg", backdrop_path "/backdrop.jpg", status "Ended", seasons with season_number 1 (poster_path "/s1poster.jpg")
- `/3/tv/1396/season/1?...` -> JSON season response: season_number 1, 2 episodes (episode 1 "Pilot", episode 2 "Cat's in the Bag")
- `/t/p/w500/poster.jpg`, `/t/p/w500/backdrop.jpg`, `/t/p/w500/s1poster.jpg` -> small JPEG bytes (Content-Type: image/jpeg)
- All other paths -> 404

Use the existing `sampleSearchJSON`, `sampleShowJSON`, `sampleSeasonJSON` constants from `client_test.go` as response fixtures where possible. If they're in a different test file (same package), they're directly accessible.

**Test helper -- `newTestTMDBMatcher(t, db)`:**
Creates a `Matcher` with:
- `db`: passed-in test DB
- `client`: `&Client{apiKey: "test-key", httpClient: srv.Client(), apiBase: srv.URL + "/3", imgBase: srv.URL + "/t/p/w500", limiter: closedChan}` where `closedChan` is a `make(chan time.Time)` that's been closed (bypasses rate limiter -- closed channel returns zero value immediately)
- `artDir`: `filepath.Join(t.TempDir(), "thumbs", "tv")`

**Test 1 -- `TestRunTVMatchIntegrationHappyPath(t *testing.T)`:**
1. Create DB via `openTestDB(t)`.
2. Seed data:
   - Insert library (type "tv", root_path "/tv")
   - Insert tv_series_files row (library_id, abs_path "/tv/breaking.bad.s01e01.mp4")
   - Insert tv_series_identities row (file_id, status "needs_fix", guessed_title "Breaking Bad", season_number 1, episode_numbers_csv "1")
   - Insert scan_jobs row (library_id, job_type "tv_match", status "running")
3. Create matcher via `newTestTMDBMatcher(t, db)`.
4. Call `m.RunTVMatch(ctx, jobID, libID)` -- should succeed.
5. Verify identity resolution:
   - Query `tv_series_identities WHERE file_id=fileID`: status should be "resolved", provider should be "tmdb", series_id should be "1396"
   - match_confidence should be > 0
6. Verify metadata caching:
   - Query `tv_series_metadata_cache WHERE entity_key='show:1396'`: should have a row with non-empty payload_json
   - Query `tv_series_metadata_cache WHERE entity_key='show:1396:season:1'`: should have a row
   - Query `tv_series_metadata_cache WHERE entity_key LIKE 'show:1396:season:1:episode:%'`: should have 2 rows (episodes 1 and 2)
7. Verify art downloads:
   - Query `tv_series_art WHERE tmdb_series_id=1396 AND art_type='series_poster'`: should have a row, and the art_path file should exist on disk
   - Query `tv_series_art WHERE tmdb_series_id=1396 AND art_type='series_backdrop'`: should have a row
   - Query `tv_series_art WHERE tmdb_series_id=1396 AND art_type='season_poster' AND season_number=1`: should have a row
8. Verify job progress: scan_jobs row should have progress_total=1 and progress_current=1.

**Test 2 -- `TestRunTVMatchIntegrationPartialFailure(t *testing.T)`:**
Test that a failed show fetch for one group does not prevent other groups from matching.
1. Create a mock server that returns 200 for search but returns 500 for `/3/tv/1396?...` (show fetch fails). Also handle a second show ID (e.g., 999) with successful responses.
2. Seed DB with two TV files with different guessed_titles:
   - File 1: guessed_title "Breaking Bad" (will match show 1396, which fails on fetch)
   - File 2: guessed_title "The Wire" (mock search returns show 999, fetch succeeds)
3. Mock server routes: search for "Breaking Bad" returns show 1396, search for "The Wire" returns show 999. Fetch for 1396 returns 500, fetch for 999 returns success.
4. Call `RunTVMatch` -- should NOT return error (per-group errors are non-fatal, just logged).
5. Verify: File 1 identity should still have status "needs_fix" (match failed but no crash). File 2 identity should be "resolved" with provider "tmdb".

Use `t.Run` subtests for logical groupings of assertions.
