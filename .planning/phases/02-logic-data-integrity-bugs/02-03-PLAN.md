<objective>
Make both music and TV scanners resilient to individual file errors (ERR-04). Currently, a hard error from ScanFile aborts the entire music scan, and hard errors in the TV scan walk abort the entire TV scan. Change both to log per-file errors and continue scanning, with an error count summary at completion.

Purpose: A single corrupt or unreadable file should not prevent the rest of a library from being scanned. Errors must be logged (not swallowed) so users can investigate, but scanning must continue.
Output: Modified scan/scanner.go and tvscan/scanner.go with continue-on-error behavior and error counting.
