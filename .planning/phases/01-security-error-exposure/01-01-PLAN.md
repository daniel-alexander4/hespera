<objective>
Harden ensureColumn against SQL injection by adding identifier validation.

Purpose: ensureColumn constructs SQL via string concatenation for table/column names (unavoidable for DDL). Without validation, a caller passing attacker-controlled strings could inject arbitrary SQL. This plan adds a compiled regexp gate at function entry.

Output: Validated ensureColumn function + comprehensive unit tests proving rejection of unsafe identifiers.
</objective>
